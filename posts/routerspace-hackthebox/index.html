RouterSpace HackTheBox

By T3sla
Posted 2022-06-27 3 min read

Bienvenidos

Este artículo tratará la resolución de la máquina RouterSpace de HackTheBox.

Información sobre la máquina

Matrix

  • APK
  • IPv6
  • SSH
  • Authorized Keys
  • Key-Gen
  • API
  • Proxy
  • RCE

Reconocimiento

Nmap (TCP)

1
2

nmap -sS -p- --open -n -v -T5 -Pn 10.10.11.148
nmap -sS -p 80,22 -n -Pn -T5 -sCV 10.10.11.148 -o content/nmap

Nmap no logra obtener ninguna versión de los servicios expuestos que están corriendo sobre la máquina.
En este caso no podríamos emplear la metodología de búsqueda de exploits públicos que afecten a una versión. Otra cosa relevante que logramos percatarnos es de la respuesta del servidor web. “Actividad sospechosa detectada”, seguramente está protegido por algún WaF o alternativa similar.

Servidor Web

Nos encontramos con una Landing Page bastante atractiva. Que trata de una Aplicación móvil para gestionar nuestro router. Analizando el HTML5 nos damos cuanta de que es estática y no obtenemos nada interesante excepto “RouterSpace.apk”.

Procedemos a realizar una petición con wget para descargar el APK.

1

wget http://10.10.11.148/RouterSpace.apk

Luego le damos una vuelta al APK, por ahora seguiremos enumerando el servidor web.

Fuzzing

Al momento de realizar la técnica fuzzing contra el servidor web no da la siguiente respuesta.

1

gobuster --url http://10.10.11.148/ -w /usr/share/dirbuster/directory-list-2.3-medium.txt -x php,txt,html

Comprobemos manualmente lo que está sucediendo.

Nuevamente, esta protección que podría ser un WaF.

APK

Retomando con el APK podríamos utilizar el comando strings y a través de grep filtrar la cadena de texto “htb” con el fin de descubrir algún dominio.

Genial obtenemos como resultado “routerspace.htb” agregamos al /etc/hosts.

Otra forma de obtener el dominio es a través de una imagen. Hay que ser observador.

Procedemos a instalar el APK en una máquina virtual con el fin de emular un dispositivo Android y así poder instalar el APK. Para luego interceptar las diferentes peticiones que realiza el APK.

Recordando que una de las funcionalidades de la aplicación es gestionar los dispositivos que están conectados a nuestro router.

Así luce la aplicación instalada, bastante atractiva la UI.

Ahora configuramos un proxy en el dispositivo móvil, con el fin de interceptar las diferentes peticiones que realizara el aplicativo móvil.

Hecho esto obtenemos el siguiente resultado.

Vemos que al presionar el botón «Check Status» realiza una petición HTTP a través del método POST.

Explotación

Podríamos intentar remplazar la dirección IP por algún comando de UNIX.

Pero no obtenemos ningún resultado, seguramente habrá algún filtro y tendremos que realizar algún tipo de bypass para conseguir que interprete los comandos.
Al momento de quitar las comillas del campo «IP» obtenemos el siguiente error.

Bypass filtro

Podemos recurrir a la utilización del operador lógico AND (&&), con el fin de poder realizar el bypass del escape de caracteres.

Genial obtenemos lo que parece ser ejecución remota de comandos.

Flag User

1

cat $HOME/user*

Lamentablemente, no podemos obtener una reverse Shell, seguramente se deba a alguna regla del iptables. Podríamos buscar algún fichero de identificación (id_rsa).

Tampoco obtenemos nada, pero tenemos permisos de escritura. Podríamos generar nosotros un fichero de identificación y luego a través del operador de comparación ”>” enviar a authorized_keys y así poder entablar una conexión mediante el servicio SSH. 

1

ssh-keygen

Ya generado nuestro fichero de identificación procedemos a copiar nuestra llave pública (id_rsa.pub) para enviar a authorized_keys.

Listo ahora nos conectamos a la máquina a través del servicio SSH.

1

ssh paul@routerspace.htb

Escalada de privilegios

Exploit

Luego de enumerar un poco el sistema GNU/Linux obtenemos algo bastante peculiar, la versión de sudo es vulnerable, además la distribución GNU/Linux es Ubuntu.

Clonamos el repositorio del exploit compilamos en nuestra máquina y luego hacemos la trasferencia de archivos.

1
2
3

git clone https://github.com/mohinparamasivam/Sudo-1.8.31-Root-Exploit.git ; cd Sudo-1.8.31-Root-Exploit.git ; make

Transferencia de archivos

Nuevamente, estamos restringidos por las reglas en cuanto a iptables.

Pero todo lo que hemos hecho hasta el momento es por IPv4, probemos por IPv6.

IPv6

Genial evadimos las reglas que estaban establecidas en el ámbito de IPv4.
